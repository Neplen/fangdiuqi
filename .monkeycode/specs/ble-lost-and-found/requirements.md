# Android BLE 防丢 APP 需求文档

## 1. 文档概述

### 1.1 项目背景

开发一款 Android BLE 防丢应用程序，用于监控用户与 iTAG 蓝牙防丢器之间的距离，当距离超出安全范围时触发报警，防止物品丢失。

### 1.2 项目目标

- 实现与 iTAG 蓝牙设备的稳定连接和通信
- 提供可靠的距离监测和防丢报警功能
- 确保应用在后台持续运行，不被关闭
- 提供友好的用户界面和 settings 配置

### 1.3 适用范围

本需求文档适用于 Android BLE 防丢 APP 的设计、开发和测试阶段。

## 2. 核心通信参数

### 2.1 支持设备

| 设备类型 | 说明 |
|---------|------|
| iTAG 防丢器 | 支持 BLE 4.0+ 的蓝牙防丢标签 |
| 通用 BLE 设备 | 支持标准 BLE 广播的其他防丢设备 |

### 2.2 BLE UUID 定义

#### 2.2.1 服务 UUID

```
连接服务 UUID: 0000ffe0-0000-1000-8000-00805f9b34fb
```

#### 2.2.2 特征值 UUID

| 特征 | UUID | 属性 | 说明 |
|------|------|------|------|
| 控制特征 | 0000ffe1-0000-1000-8000-00805f9b34fb | WRITE | 发送控制命令到设备 |
| 通知特征 | 0000ffe1-0000-1000-8000-00805f9b34fb | NOTIFY/INDICATE | 接收设备通知 |

### 2.3 控制协议

#### 2.3.1 命令格式

```
命令长度：1-2 字节
字节序：小端序 (Little Endian)
```

#### 2.3.2 控制命令表

| 命令 | 字节值 | 说明 |
|------|--------|------|
| 拍照 | 0xFF | 触发手机拍照 |
| 静音 | 0xFE | 关闭设备蜂鸣 |
| 连接请求 | 0x01 | 主动连接设备 |
| 断开请求 | 0x00 | 断开设备连接 |

#### 2.3.3 设备响应

| 响应类型 | 字节值 | 说明 |
|---------|--------|------|
| 按键按下 | 0x01 | 用户按下 iTAG 按键 |
| 电量低 | 0x02 | 设备电量低警告 |
| 连接成功 | 0x03 | 设备连接建立 |
| 断开连接 | 0x04 | 设备连接断开 |

## 3. 防丢逻辑

### 3.1 RSSI 距离判断

#### 3.1.1 信号强度分级

| 等级 | RSSI 范围 (dBm) | 距离估计 | 状态 |
|------|----------------|----------|------|
| 极近 | > -50 | 0-1 米 | 安全 |
| 近 | -50 ~ -65 | 1-3 米 | 安全 |
| 中 | -65 ~ -75 | 3-5 米 | 警告 |
| 远 | -75 ~ -85 | 5-10 米 | 危险 |
| 极远 | < -85 | > 10 米 | 断开 |

#### 3.1.2 距离阈值配置

```
默认报警阈值：-75 dBm
滞后区间：5 dBm (防止信号波动导致频繁报警)
采样间隔：1000ms
连续触发次数：3 次 (连续 3 次超出阈值才触发报警)
```

#### 3.1.3 算法要求

- 使用滑动平均滤波处理 RSSI 值
- 实现信号滞后比较器，避免临界值抖动
- 支持用户自定义报警阈值

### 3.2 WIFI 勿扰模式

#### 3.2.1 功能描述

当手机连接到指定的 WiFi 网络时，自动暂停防丢监控，避免在家庭、办公室等安全区域误报。

#### 3.2.2 配置项

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 勿扰开关 | 启用/禁用功能 | 关闭 |
| 信任 WiFi 列表 | 用户配置的 SSID 列表 | 空 |
| 自动添加 | 当前连接的 WiFi 可一键加入 | - |

#### 3.2.3 行为规则

```
IF WiFi 连接状态 == 已连接
   AND 当前 SSID IN 信任列表
THEN
   暂停 RSSI 监控
   显示"勿扰模式"状态
   不触发报警
ELSE
   恢复 RSSI 监控
   显示正常状态
```

### 3.3 报警铃声

#### 3.3.1 报警类型

| 类型 | 触发条件 | 铃声特性 |
|------|----------|----------|
| 距离报警 | RSSI 低于阈值 | 持续响铃，音量 100% |
| 断开报警 | 设备断开连接 | 急促提示音，3 次 |
| 电量报警 | 设备电量低 | 温和提示音，1 次 |

#### 3.3.2 铃声配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 铃声选择 | 从系统铃声库选择 | 默认报警音 |
| 自定义铃声 | 支持用户导入音频文件 | - |
| 音量 | 报警时音量 (0-100%) | 100% |
| 振动 | 报警时是否振动 | 开启 |
| 重复次数 | 报警音重复次数 | 持续 |

#### 3.3.3 消音操作

- 点击通知栏闹钟可静音
- 按下设备按键可远程静音
- APP 内提供手动停止按钮

### 3.4 定时勿扰

#### 3.4.1 功能描述

在用户设定的时间段内，暂停防丢报警功能，适用于夜间睡眠等场景。

#### 3.4.2 配置项

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 勿扰开关 | 启用/禁用功能 | 关闭 |
| 开始时间 | 勿扰模式开始时刻 | 22:00 |
| 结束时间 | 勿扰模式结束时刻 | 07:00 |
| 重复周期 | 每天/工作日/自定义 | 每天 |

#### 3.4.3 行为规则

```
IF 当前时间 IN [开始时间，结束时间]
   AND 勿扰模式 == 开启
THEN
   暂停报警触发
   保持 BLE 连接
   记录断开事件但不报警
ELSE
   正常监控
```

### 3.5 自动重连

#### 3.5.1 重连策略

| 场景 | 重连间隔 | 最大重试次数 | 超时时间 |
|------|----------|-------------|----------|
| 正常断开 | 5 秒 | 无限 | 每次 30 秒 |
| 报警后断开 | 3 秒 | 10 次 | 每次 15 秒 |
| 后台运行 | 30 秒 | 无限 | 每次 60 秒 |

#### 3.5.2 重连流程

```
1. 检测设备断开事件
2. 检查是否在勿扰模式 (WiFi/定时)
3. 如果是勿扰模式，记录状态，延迟重试
4. 如果不是勿扰模式，立即开始重连
5. 按重连间隔扫描设备
6. 发现设备后发起连接
7. 连接成功后恢复监控
8. 达到最大重试次数后，发送通知并暂停
```

#### 3.5.3 用户通知

- 重连中：状态栏显示"正在重连..."
- 重连失败：推送通知"设备已超出范围"
- 重连成功：显示"设备已重新连接"

## 4. 后台保活

### 4.1 前台服务 (Foreground Service)

#### 4.1.1 服务类型

```
服务类型：FOREGROUND_SERVICE_TYPE_LOCATION + FOREGROUND_SERVICE_TYPE_CONNECTED_DEVICE
通知渠道：ble_monitor_channel
通知优先级：PRIORITY_LOW
```

#### 4.1.2 通知内容

| 元素 | 内容 |
|------|------|
| 标题 | "BLE 防丢监控中" |
| 设备名称 | "已连接：iTAG-XXXX" |
| 信号强度 | "信号强度：-65 dBm" |
| 操作按钮 | [设置] [停止监控] |

#### 4.1.3 保活要求

- Android 8.0+ 必须使用前台服务
- 服务在应用启动后自动开启
- 用户可手动停止服务
- 服务被杀后自动重启 (onStartCommand 返回 START_STICKY)

### 4.2 WakeLock 管理

#### 4.2.1 锁类型

```
锁类型：PARTIAL_WAKE_LOCK
用途：保持 CPU 运行，确保后台扫描正常
```

#### 4.2.2 获取时机

- 前台服务启动时获取 WakeLock
- BLE 扫描期间保持持有
- 扫描间隙可释放以省电

#### 4.2.3 释放时机

- 用户手动停止监控
- 应用被 Force Stop
- onDestroy 回调中必须释放

### 4.3 权限引导

#### 4.3.1 必需权限清单

| 权限 | 用途 | 申请时机 |
|------|------|----------|
| BLUETOOTH | BLE 基础功能 | 首次启动 |
| BLUETOOTH_ADMIN | BLE 设备管理 | 首次启动 |
| ACCESS_FINE_LOCATION | BLE 扫描 (Android 12-) | 首次连接 |
| ACCESS_COARSE_LOCATION | BLE 扫描 (Android 12-) | 首次连接 |
| BLUETOOTH_SCAN | BLE 扫描 (Android 12+) | 首次连接 |
| BLUETOOTH_CONNECT | BLE 连接 (Android 12+) | 首次连接 |
| POST_NOTIFICATIONS | 通知推送 (Android 13+) | 首次启动 |
| FOREGROUND_SERVICE | 前台服务 | 首次启动 |
| REQUEST_IGNORE_BATTERY_OPTIMIZATIONS | 电池优化豁免 | 设置页引导 |

#### 4.3.2 权限申请流程

```
1. 应用启动
   ↓
2. 检查必要权限
   ↓
3. 未授权权限 → 显示权限说明弹窗
   ↓
4. 请求运行时权限
   ↓
5. 权限被拒绝 → 引导至设置页手动开启
   ↓
6. 全部授权 → 继续正常流程
```

#### 4.3.3 电池优化豁免引导

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 进入设置页 | 点击"电池优化"选项 |
| 2 | 显示引导弹窗 | 说明豁免的必要性 |
| 3 | 跳转系统设置 | 使用 ACTION_REQUEST_IGNORE_BATTERY_OPTIMIZATIONS |
| 4 | 用户选择"不允许" | 引导手动在系统设置中操作 |
| 5 | 返回应用检查状态 | 确认是否已豁免 |

### 4.4 后台限制应对

#### 4.4.1 厂商适配

针对以下厂商进行特殊适配：

| 厂商 | 适配措施 |
|------|----------|
| 小米 (MIUI) | 引导开启"自启动"和"后台运行"权限 |
| 华为 (EMUI) | 引导加入"受保护应用"名单 |
| OPPO (ColorOS) | 引导开启"允许后台活动" |
| VIVO (Funtouch) | 引导开启"后台高耗电" |
| 三星 (OneUI) | 引导关闭"休眠未使用应用" |

#### 4.4.2 保活策略

- 使用 JobScheduler 定期唤醒
- 监听 BOOT_COMPLETED 开机自启
- 使用 AlarmManager 设置定期心跳

## 6. 非功能性需求

### 6.1 性能要求

| 指标 | 要求 |
|------|------|
| 连接建立时间 | < 3 秒 |
| RSSI 刷新频率 | 1 次/秒 |
| 报警响应延迟 | < 2 秒 |
| 后台 CPU 占用 | < 5% |
| 日均耗电量 | < 10% |
| 内存占用 | < 50 MB |

### 6.2 兼容性要求

| 项目 | 要求 |
|------|------|
| 最低 Android 版本 | Android 8.0 (API 26) |
| 目标 Android 版本 | Android 14 (API 34) |
| BLE 版本 | BLE 4.0+ |
| 屏幕适配 | 支持所有主流分辨率 |
| 语言 | 简体中文、英文 |

### 6.3 安全要求

- 不收集用户隐私数据
- 位置数据仅本地存储
- 不包含第三方广告 SDK
- 符合 Google Play 隐私政策

### 6.4 可用性要求

- 首次启动提供引导流程
- 关键操作提供确认提示
- 错误场景提供友好提示
- 支持无障碍访问 (TalkBack)
