
================================================================================
修改说明
================================================================================

【修改文件1】BleManager.kt

1. 新增常量：
   - RECONNECT_COOLDOWN_MS = 5000L（重连冷却时间5秒）

2. 新增属性：
   - isConnecting: Boolean（防重复连接锁）
   - lastDisconnectTime: Long（记录上次断开时间）

3. 修改 onConnectionStateChange() - STATE_CONNECTED：
   - 添加 isConnecting = false（连接成功释放锁）

4. 修改 onConnectionStateChange() - STATE_DISCONNECTED：
   - 添加 lastDisconnectTime = System.currentTimeMillis()
   - 添加 isConnecting = false（释放连接锁）
   - 重连延迟改为动态计算（确保至少冷却5秒）
   - 删除 gatt.close() 前的 "释放原生资源" 误导日志

5. 修改 connectDirectly()：
   - 入口增加 isConnecting 检查（防重复连接）
   - 增加已连接同设备检查（避免重复连接）
   - 设置 isConnecting = true 标记连接中
   - 删除 cleanupGatt() 调用，改为内联安全清理
   - 异常时设置 isConnecting = false

6. 修改 scheduleReconnect()：
   - 增加 isConnecting 检查
   - 设置 isConnecting = true
   - 异常时设置 isConnecting = false

7. 修改 reconnectIfDisconnected()：
   - 增加冷却时间检查（5秒内不重连）
   - 增加 isConnecting 检查
   - 删除 cleanupGatt() 调用，改为内联安全清理

8. 修改 cleanupGatt()：
   - 删除所有反射代码（mBluetoothGattField, mCallback）
   - 只保留标准 API：disconnect() + close()
   - 添加 isConnecting = false

【修改文件2】BleScanner.kt

1. 新增属性：
   - scanJob: Job?（扫描任务引用）

2. 修改 startScan()：
   - 启动扫描前检查 isScanning，如果正在进行则先 stopScan() 并延迟500ms
   - 添加 "扫描已启动" 日志
   - 异常时设置 isScanning = false

3. 修改 awaitClose：
   - 添加 "扫描已停止" 日志

4. 修改 stopScan()：
   - 重新获取蓝牙适配器，确保使用最新 scanner 实例

================================================================================
修改原则
================================================================================

- 只修改蓝牙连接和扫描的底层问题
- 不触及任何报警逻辑、WiFi逻辑、出门提醒逻辑
- 删除破坏性的反射代码，使用标准API
- 增加防重复机制，避免资源耗尽
